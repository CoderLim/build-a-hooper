过去两天我们上线了自站点推出以来最大的一波「玩法周边」更新：**账号体系、累积型 Legacy 排行榜，以及从「随便玩玩」到「保存成绩」的更清晰路径。** 下面是改动摘要与体验方式。

## 先玩再登录，不挡路

Build a Hooper 仍然是 **游客优先** 的设计。不注册也能完整体验：选模式、组 build、模拟赛季、到达 **My Card**。

登录带来的核心收益只有一个：**完成的赛季可以保存到云端，并计入排行榜。**

|                         | 游客   | 已登录 |
| ----------------------- | ------ | ------ |
| 完整游戏流程            | 可以   | 可以   |
| 本地续玩（同一浏览器）  | 可以   | 可以   |
| 分享 My Card / 再来一局 | 可以   | 可以   |
| 赛季结束后保存 run      | 不可以 | 可以   |
| Legacy 积分与排名       | 不可以 | 可以   |

未登录完成赛季时，My Card 会提示登录以便提交成绩；卡片展示、分享、重开等功能不受影响。

## Legacy 排行榜 —— 看长期，不看单局

新的 [/leaderboard](/leaderboard) 按 **Legacy** 排名 —— 这是所有已保存 run 累积出的长期分数，而不是「某一局打得最好」的单次榜。

支持多种排序：

- **Legacy 积分**（默认）
- **总冠军数**
- **胜率**
- **最高 Overall**
- **完成 run 数**
- **奖项总数**

点击玩家可进入 **个人资料页**，查看生涯式汇总。榜单鼓励长期投入：稳定刷 run、冲冠军、或反复试验 build 的玩家都有追求空间。

## 成绩如何保存

登录后完成赛季并进入 **My Card** 时：

1. 客户端根据模式、位置、build 槽位和赛季数据组装 run payload；
2. 请求 `POST /api/hooper/runs`；
3. 服务端写入 run，并更新你的 **Legacy** 记录（积分、胜场、冠军等）。

同一 run 在浏览器内会去重，避免刷新 My Card 重复提交。

## 导航与游戏页统一

[/game](/game) 现已与首页共用同一套顶栏 —— **Play**、**Leaderboard**、指南链接，以及未登录时的 **登录 / 注册**。

游戏内顶栏更精简（主要保留 **Restart**），减少重复导航占用的纵向空间。

## 登录体验修复

测试阶段反馈的两个问题已处理：

**不再跳进 Settings 后台。** 注册或登录后会回到来源页（首页、游戏页或排行榜），而不是 SaaS 模板自带的账户设置页。

**生产环境 Invalid origin。** 修复了 [buildahooper.org](https://buildahooper.org) 上可能出现的 `Invalid origin` 注册错误（部署域名与 auth 配置不一致导致）。目前生产站注册、登录应可正常使用。

## 先 Staging，再上线

新功能先在独立的 **staging** Worker 与数据库验收，确认无误后再更新 [buildahooper.org](https://buildahooper.org)，避免未测内容直接影响线上玩家。

## 怎么试

1. [开一局](/game) —— 无需账号；
2. 想上榜时 [注册](/sign-up)；
3. 打完赛季后打开 [/leaderboard](/leaderboard) 看排名。
