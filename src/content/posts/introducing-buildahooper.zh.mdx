**Build a Hooper** 是一款非官方浏览器篮球模拟器。你可以从传奇与现代球队风格中搭配技能，在 **13 项属性** 上塑造自定义球员，并挑战完整 **82 场赛季** —— 全程在浏览器里完成，无需安装。

如果你是从搜索、朋友推荐或社交媒体第一次看到这个名字，想先搞清楚这款游戏到底是什么，这篇文章可以作为你第一次 draft 之前的入门阅读。

## Build a Hooper 是什么？

Build a Hooper 把「建队幻想」压缩成一个轻量、可反复试验的模拟循环。你不是只看一场高光，而是在验证一个想法：**什么样的球员能长期赢球？**

核心循环可以分成三步：

1. **确定方向** —— 选择能形成清晰身份的技能，而不是堆一堆互不相关的加成。
2. **围绕属性构建** —— 13 项属性并存时，协同往往比单项数值更重要。
3. **撑过赛季** —— 82 场赛程奖励的是稳定性，而不只是偶尔的爆发。

每一轮都是一次小型篮球实验。乐趣从 draft 开始，但深度会在后面出现 —— 你会发现每一次技能选择都会改变你该如何看待属性，也会改变这套 build 能承受多大的风险。

## 三种模式，三种谜题

游戏提供 **三种模式** —— Classic、Blind、Chaos —— 每种都会改变 build 阶段你能看到什么、能提前规划多少。

- **Classic** —— 最适合入门，信息最完整，适合围绕清晰身份规划 build。
- **Blind** —— build 阶段隐藏更多信息，逼你更早做出方向选择。
- **Chaos** —— 最不可预测的环境，适合检验一套 build 思路在失控条件下是否还成立。

不要假设一套成功 build 可以原封不动地通吃所有模式。把每种模式都当成新的篮球环境，相应调整自己的思路。

## 属性如何塑造你的 build

13 项属性并存时，没有哪一项能独自扛起一切。更强的 run 往往来自多项优势彼此支撑，而不是某一个夸张数值。

如果你的 draft 偏进攻爆发，仍然需要足够平衡来撑过漫长赛季；如果更偏向全面稳定，目标就变成最大化可重复产出，而不是追逐最戏剧性的峰值。

把终结、组织、防守、身体条件和决策能力看成同一身份的不同部分，而不是孤立升级。当两三项属性共同支撑同一个角色时，你的球员会更容易读懂，也更容易在多次尝试中持续改进。

## 为什么 82 场赛季很重要

赛季结构是这款游戏幻想感的核心。漫长赛程会把目标从「瞬间爽感」拉回到「长期可重复表现」。

真正强的赛季 build，通常不只会打巅峰局，而是连普通夜晚也有明确产出逻辑。可以问自己：到了第 40 场、第 60 场、第 82 场，这套选择还合理吗？它能不能从一段低迷里恢复？当对位暴露短板时，是否还有足够平衡去撑住？

Build a Hooper 里最好的球员，往往不一定最夸张，而是优势能持续到赛季后段的那一个。

## 如何开始？

1. 前往 [/game](/game) 或 [首页](/)，点击 **立即游玩**。
2. 选择适合当下心情的模式 —— Classic 信息最全，Blind 或 Chaos 则更像解谜。
3. 每轮只验证一种球员身份；赛季结束后，先问「最先崩的是哪一环」，再决定是否重开。
4. 需要系统建议时，回到首页阅读完整攻略，了解属性、模式与 82 场赛季的思路。
